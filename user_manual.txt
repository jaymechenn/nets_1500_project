WikiPath - User Manual

Our project finds the shortest hyperlink path between two Wikipedia articles
using breadth-first search. Articles are nodes, hyperlinks are directed
edges. You can run it as a text menu in a terminal, or as a browser UI.


Requirements
- Java 17 or newer.
- A modern browser (for the web UI).
- Use -Xmx4g if running the full Wikipedia graph.


Build
    mkdir -p target/classes
    javac --release 17 -d target/classes src/main/java/wikipath/*.java


Run with the demo graph (18 articles)
    java -cp target/classes wikipath.Main demo-data/edges.txt demo-data/names.txt


Run with the full Stanford SNAP graph (~1.79M articles)
    bash scripts/download-data.sh
    java -Xmx4g -cp target/classes wikipath.Main \
         data/wiki-topcats.txt data/wiki-topcats-page-names.txt


Text menu
After the graph loads you see:

    WikiPath Menu
    1. Find shortest path between two articles
    2. Run random pair analysis (also prints top bridge articles)
    3. Exit

Option 1 asks for a source and target. You can type a title
(case-insensitive, spaces and underscores are equivalent) or a numeric
article ID. It prints the path, the number of hops, and the time taken.

Option 2 asks how many random article pairs to sample. It runs BFS for
each pair and prints the average path length, clustering and overlap
measures, and the top 10 bridge articles seen across the sample.

Option 3 exits.


Browser UI
Add --serve to start a local server, then open http://localhost:8080:

    java -cp target/classes wikipath.Main --serve \
         demo-data/edges.txt demo-data/names.txt

For the full graph:

    java -Xmx4g -cp target/classes wikipath.Main --serve \
         data/wiki-topcats.txt data/wiki-topcats-page-names.txt


The page has a sidebar (controls) and a main pane (results). The header
shows a green "live backend" tag when the Java server is running, and an
amber "demo (static)" tag if the page falls back to the small graph.json
file.


Find a path
1. Type into "Source article". A dropdown suggests matching titles after
   you have typed at least one letter. Pick one with the mouse, or with
   Up/Down + Enter.
2. Do the same for "Target article".
3. Click "Find path" or press Enter.

The result shows up two ways:
- A diagram of the path (large blue circles for the path nodes, small
  grey circles for a few of each path node's neighbors).
- A row of clickable pills with the article titles. Each pill opens the
  real Wikipedia article in a new tab.

Stats: hops, time in milliseconds, articles in the path.


Other buttons
- Random pair: picks two random articles and runs BFS automatically.
- Swap: swaps source and target.
- Quick examples: prefilled pairs you can click to try.


Random-pair analysis
In the sidebar, set a sample size (1-5000) and click "Run analysis". An
Analysis panel appears with seven stat cards (trials, reachable, average
path length, clustering, overlap, local bridges, time) and a ranked list
of the top bridge articles with bar charts. Each bridge article is also
a Wikipedia link.
